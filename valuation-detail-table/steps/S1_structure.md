### Step 1.2a D1/D2/D3映射策略 [DT-119]

> **⚠️ 本步骤为v3.29新增，补充评估明细表填写的大逻辑框架**

科目映射表建立时，MUST按D1/D2/D3三级递进顺序组织映射关系：

**映射策略表**：

| 映射层级 | 映射方向 | 映射内容 | 异常处理 |
|---------|---------|---------|---------|
| **D1→D2** | BS科目→科目余额表编码 | 货币资金→1001+1002+1012；应收账款→1122；... | DT-117(无编码→差额推论) / DT-118(汇总数≠BS→重分类) |
| **D2→D3** | 科目余额表末级科目→辅助数据源 | 1002→银行对账单PDF；1122→辅助余额表；1601→卡片台账 | DT-111(辅助余额表强制提取) / DT-117(D2无编码但D3有数据→待确认映射) |

**关键约束**：
1. **D1为准**：明细表合计MUST与BS一致（DT-119-2）。科目余额表金额仅是中间桥梁，最终勾稽目标是BS
2. **先D1-D2再D2-D3**：不允许跳过D2直接从D1映射D3（DT-119-1），否则编码体系信息和重分类线索丢失
3. **D2负数项目必须检查重分类**：科目余额表中每个往来科目的负数子项，MUST按DT-118重分类映射表检查（DT-119-3）

### Step 1.3 确定各科目填写策略

| 科目类型 | 填写策略 | 数据来源 |
|---------|---------|---------|
| **货币资金** | 银行存款按账户逐行填；现金/其他货币资金按余额填 | 科目余额表+序时账 |
| **往来类（应收/应付/预收/预付/其他应收应付）** | 按结算对象逐行填写 | 科目余额表明细+序时账 |
| **存货类** | 按品种/类别汇总行填写 | 科目余额表+收发存明细 |
| **固定资产** | 按设备类别汇总行填写 | 科目余额表+固定资产台账 |
| **无形资产** | 按权属逐项填写 | 科目余额表+权属资料 |
| **借款类** | 按借款合同逐行填写 | 科目余额表+借款合同 |
| **减值准备** | 填入对应科目"减："行 | 科目余额表坏账准备/存货跌价准备科目 |

## 附录B：评估明细表列定义速查

### 资产类明细表

> ⚠️ **资产类与负债类的C/D列顺序相反！** 资产类：C=业务内容(文字), D=发生日期(日期)；负债类：C=发生日期(日期), D=业务内容(文字)。编辑前必须核对表头！

| 列 | 含义 | 数据类型 |
|----|------|---------|
| A | 序号 | 数字 |
| B | 户名/名称/结算对象 | 文本 |
| C | **业务内容**/规格型号 | **文字**，禁止填日期序列号 |
| D | **发生日期**/购置日期 | **日期**（datetime对象），禁止填文字 |
| E | 币种 | "人民币"(宋体) |
| F | 外币账面金额/账面原值 | 数值 |
| G | 评估基准日汇率/账面价值 | 数值 |
| H | 账面价值/评估价值 | 数值(会计格式) |
| I | 评估价值/增值额 | 数值(会计格式) |
| J | 增值额/增值率% | 公式=I-H |
| K | 增值率%/备注 | 公式=IF(H=0,"",J/H*100) |
| L | 备注 | 文本 |

### 负债类明细表

> ⚠️ **负债类与资产类的C/D列顺序相反！** 负债类：C=发生日期(日期), D=业务内容(文字)。

| 列 | 含义 | 数据类型 |
|----|------|---------|
| A | 序号 | 数字 |
| B | 户名/结算对象 | 文本 |
| C | **发生日期** | **日期**，禁止填文字 |

| E | 币种 | "人民币"(宋体) |
| F | 外币账面金额 | 数值 |
| G | 评估基准日汇率 | 数值 |
| H | 账面价值 | 数值(会计格式) |
| I | 评估价值 | 数值(会计格式) |
| J | 增值额 | 公式=I-H |
| K | 增值率% | 公式=IF(H=0,"",J/H*100) |
