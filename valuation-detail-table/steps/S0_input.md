### Step 0.3 解析资产负债表

提取资产负债表各科目期末余额，用于后续勾稽核对。

**🚨 DT-139 BS解析后强制自校验（MUST执行，assert断言）**：

> **根因**：河南平绿项目bs_balances.json中G/H列值全部映射到错误的A列项目名（4列格式BS的行偏移），DT-119假设D1(BS)是"权威基准"但D1自身解析错误→全链路污染→所有勾稽都从错误基准出发。本规则要求BS解析后立即自校验，防止错误数据流入下游。

**4列格式BS解析注意事项**：
- BS可能为4列格式：A=项目名, C=年初, D=年末(资产侧), G=年初, H=年末(负债侧)
- **G/H列与A列项目名可能存在行偏移**——G/H列数据可能比A列项目名偏移若干行
- MUST通过"项目名+金额"双重匹配确认行对齐，禁止仅凭行号匹配

**自校验脚本（MUST在BS解析脚本末尾执行）**：
```python
# ╔══════════════════════════════════════════════════════════════════╗
# ║ DT-141: 规则校验清单（BS解析脚本，脚本自检=规则执行确认）      ║
# ╚══════════════════════════════════════════════════════════════════╝
#
# [DT-0]  零幻觉：BS数据从文件提取 → 下方bs_items从bs_balances.json加载
# [DT-139] BS解析后强制自校验 → 下方3项assert断言
# ═════════════════════════════════════════════════════════════════

# DT-139: BS解析后强制自校验
bs_items = bs_data['data']['items']  # 从bs_balances.json加载

# 校验1: 资产总计 - 负债合计 - 权益合计 ≤ 1元（BS会计等式）
total_assets = bs_items.get('资产总计', 0) or bs_items.get('资产合计', 0)
total_liabilities = bs_items.get('负债合计', 0) or bs_items.get('负债总计', 0)
total_equity = bs_items.get('所有者权益合计', 0) or bs_items.get('权益合计', 0)
balance_diff = abs(total_assets - total_liabilities - total_equity)
assert balance_diff <= 1, \
    f"CRITICAL: BS会计等式不等! 资产总计={total_assets:,.2f} - 负债={total_liabilities:,.2f} - 权益={total_equity:,.2f} = 差{balance_diff:,.2f}"
print(f"[DT-139] ✅ 校验1通过: 资产=负债+权益 (差{balance_diff:.2f})")

# 校验2: 流动资产合计 = 子项之和 ±1元
current_assets_items = ['货币资金', '应收账款', '预付款项', '其他应收款', '存货', '其他流动资产']
current_assets_sum = sum(bs_items.get(item, 0) or 0 for item in current_assets_items)
current_assets_total = bs_items.get('流动资产合计', 0) or bs_items.get('流动资产', 0)
if current_assets_total > 0:
    diff2 = abs(current_assets_total - current_assets_sum)
    assert diff2 <= 1, \
        f"CRITICAL: 流动资产合计≠子项之和! 合计={current_assets_total:,.2f}, 子项之和={current_assets_sum:,.2f}, 差={diff2:,.2f}"
    print(f"[DT-139] ✅ 校验2通过: 流动资产合计=子项之和 (差{diff2:.2f})")

# 校验3: 流动负债合计 = 子项之和 ±1元
current_liab_items = ['短期借款', '应付账款', '预收款项', '应付职工薪酬', '应交税费', '其他应付款', '其他流动负债']
current_liab_sum = sum(bs_items.get(item, 0) or 0 for item in current_liab_items)
current_liab_total = bs_items.get('流动负债合计', 0) or bs_items.get('流动负债', 0)
if current_liab_total > 0:
    diff3 = abs(current_liab_total - current_liab_sum)
    if diff3 > 1:
        print(f"[DT-139] ⚠️ 流动负债合计≠子项之和: 合计={current_liab_total:,.2f}, 子项之和={current_liab_sum:,.2f}, 差={diff3:,.2f}")
        # 不assert中断，因为可能有合同负债等未在列表中的项目
    else:
        print(f"[DT-139] ✅ 校验3通过: 流动负债合计=子项之和 (差{diff3:.2f})")

# 校验4: G/H列行对齐校验（4列格式BS特有）
# 遍历BS sheet中A列和G/H列，确认金额能匹配上正确的项目名
print(f"[DT-139] ✅ BS解析自校验完成")

# ═════════════════════════════════════════════════════════════════
# DT-141: 规则执行汇总确认（BS解析脚本末尾自检）
# ═════════════════════════════════════════════════════════════════
rules_checked = {
    'DT-0': bs_items is not None and len(bs_items) > 0,
    'DT-139': True,  # 自校验已通过（否则上方assert已crash）
}
failed = [k for k, v in rules_checked.items() if not v]
assert len(failed) == 0, f"CRITICAL [DT-141]: BS解析规则执行未确认: {failed}"
print(f"[DT-141] ✅ BS解析规则执行汇总: {len(rules_checked)}条规则全部确认")
```

**校验失败处理**：
- 校验1（资产≠负债+权益）失败=CRITICAL=BS解析不可靠→禁止进入Phase 1→MUST回退重新解析BS
- 校验2/3失败=WARNING→检查是否有项目名匹配遗漏→补充后重校验
- 校验4未实现自动化时，MUST人工确认G/H列行对齐

### Step 0.4 🚨 PDF数据源回填校验 [DT-109]

> **⚠️ 本步骤为v3.14新增硬约束，河南平绿项目复盘根因修复**
> **根因**：河南平绿项目Phase 0解析科目余额表后发现1002银行存款仅1行汇总，但24份银行对账单PDF从未被Phase 2引用。科目余额表1002子科目数=1≠银行对账单账户数=16+，DT-104仅检查子科目数下限，未检查PDF数据源是否被实际使用。

**核心问题**：Phase -1的DT-108确保了PDF被提取，但Phase 2仍可能跳过PDF数据（如科目余额表有汇总数就直接用汇总数填写）。需要一个从"科目→数据源"的反向校验。

**校验规则**：

```
Phase 0 解析科目余额表后
  │
  ├── 对每个有余额的关键科目，确认其数据源
  │     │
  │     ├── 科目余额表有末级子科目 → 优先使用子科目数据填写
  │     │
  │     ├── 科目余额表无末级子科目，但有对应PDF/辅助数据源
  │     │     │
  │     │     ├── PDF已提取（DT-108 PASS） → MUST使用PDF数据填写，禁止用汇总数
  │     │     │
  │     │     └── PDF未提取或提取失败 → 可用汇总数填写，但MUST标注"PDF数据源缺失"
  │     │
  │     └── 科目余额表无末级子科目，也无对应PDF/辅助数据源
  │           → 用汇总数填写，标注"无可细分数据源"
  │
  └── 生成科目→数据源映射表
```

**科目→数据源强制映射表**：

| 科目编码 | 科目名称 | 首选数据源 | 次选数据源 | 回填校验 |
|----------|---------|-----------|-----------|---------|
| 1002 | 银行存款 | 银行对账单PDF | 科目余额表子科目 | PDF有数据时禁止用汇总数 |
| 1004 | 其他货币资金 | 保证金对账单PDF | 科目余额表子科目 | PDF有数据时禁止用汇总数 |
| 1123 | 预付款项 | 辅助余额表 | 科目余额表子科目 | 辅助表有数据时禁止用汇总数 |
| 1221 | 其他应收款 | 辅助余额表 | 科目余额表子科目 | 同上 |
| 1601 | 固定资产 | 卡片台账PDF | 科目余额表子科目 | PDF有数据时禁止用汇总数 |
| 2202 | 应付账款 | 辅助余额表 | 科目余额表子科目 | 辅助表有数据时禁止用汇总数 |
| 2241 | 其他应付款 | 辅助余额表 | 科目余额表子科目 | 同上 |

**DT-109核心约束**：
- **DT-109-1**：当PDF/辅助数据源已被成功提取（DT-108 PASS），Phase 2填写对应科目时MUST使用PDF/辅助数据源的明细数据，**禁止用科目余额表汇总数代替**。违反=CRITICAL。
- **DT-109-2**：当科目余额表有末级子科目（如1002下有16个银行账户），3-1-2 Sheet行数MUST≥子科目数。当科目余额表无末级子科目但有PDF对账单，3-1-2 Sheet行数MUST≥对账单账户数。两者取MAX作为填写行数下限。
- **DT-109-3**：Phase 2填写完成后，必须反向校验：每个已被提取的PDF文件，其数据是否被实际引用到了对应Sheet。未被引用=CRITICAL。

### Step 0.5a 🚨 D1/D2/D3三级递进映射建立 [DT-119]

> **⚠️ 本步骤为v3.29新增硬约束，补充评估明细表填写的大逻辑框架**
> **核心原则**：各科目数据应当以资产负债表(D1)为准。填写时，MUST先建立D1与D2的映射关系，再建立D2与D3的映射关系。不允许跳级映射。

**三级数据源定义**：

| 层级 | 数据源 | 角色 | 内容 |
|------|--------|------|------|
| **D1** | 资产负债表 | **权威基准**（最终勾稽目标） | 各科目期末余额，编制时已包含重分类调整 |
| **D2** | 科目余额表 | **编码桥梁**（科目编码→末级科目→期末余额） | 按会计科目编码体系组织的借贷方余额明细，含负数项目等重分类线索 |
| **D3** | 辅助明细账/对账单/各类台账和末级明细表格 | **末级明细**（逐行数据） | 银行对账单、辅助余额表、固定资产卡片台账、保证金对账单、收发存明细等 |

**执行步骤**：

#### 0.5a.1 建立D1→D2映射（BS科目→科目余额表编码）

#### 0.5a.2 建立D2→D3映射（科目余额表末级科目→辅助数据源）

对科目余额表中每个有余额的末级科目，确认其D3明细数据源：

def extract_auxiliary_balance(file_path):
    """提取辅助余额表中的结算对象和期末余额"""
    wb = openpyxl.load_workbook(file_path, data_only=True)

    results = []

    for r in range(1, 11):
        row_vals = [ws.cell(row=r, column=c).value for c in range(1, ws.max_column+1)]

    # 典型列结构：A=科目编码, B=科目名称, C=结算对象编码,
    #             D=结算对象名称, ... , I/J=期末借方/贷方
    name_col = None  # 结算对象名称列
    debit_col = None  # 期末借方余额列
    credit_col = None  # 期末贷方余额列

    for r in range(1, 6):
        for c in range(1, ws.max_column + 1):
            val = ws.cell(row=r, column=c).value
            if val and '结算对象' in str(val) and '名称' in str(val):
                name_col = c
            elif val and '期末' in str(val) and '借方' in str(val):
                debit_col = c
            elif val and '期末' in str(val) and '贷方' in str(val):
                credit_col = c

    # 提取每个结算对象（排除合计行）
    for r in range(data_start_row, ws.max_row + 1):
        name = ws.cell(row=r, column=name_col).value
        if name and '合计' not in str(name) and '小计' not in str(name):
            debit = ws.cell(row=r, column=debit_col).value or 0
            credit = ws.cell(row=r, column=credit_col).value or 0
            results.append({
                'name': str(name).strip(),
                'debit': float(debit) if debit else 0,
                'credit': float(credit) if credit else 0,
                'balance': float(credit) - float(debit) if credit else -float(debit)
            })

    wb.close()
    return results

### Step 0.7 🚨 设定信息页提前填写 [DT-121]

> **⚠️ 本步骤为v3.31新增硬约束，用户要求"设定信息页需要根据财务报表填写被评估单位和评估基准日；以上步骤在正式填写明细表之前"**

3. **避免遗忘**：Phase 5交付前任务繁重（隐藏/格式/COM重算/页脚等），设定信息容易遗漏；提前到Phase 0可确保信息提取即填写

#### 0.7.1 提取被评估单位全称

```python
import openpyxl, re

def extract_company_name(bs_path, kmye_path=None):
    """从财务报表中提取被评估单位全称"""
    # 优先级1：BS表头标题行
    wb = openpyxl.load_workbook(bs_path, data_only=True)
    ws = wb.active
    for r in range(1, 6):  # 通常在前5行
        cell_val = ws.cell(row=r, column=1).value
        if cell_val:
            # 典型格式："资产负债表" 或 "河南平煤神马平绿置业有限公司资产负债表"
            # 或单独一行 "编制单位：河南平煤神马平绿置业有限公司"
            text = str(cell_val).strip()
            # 模式1：编制单位：XXX
            m = re.search(r'编制单位[：:]\s*(.+?)(?:\s|$)', text)
            if m:
                wb.close()
                return m.group(1).strip()
            # 模式2：公司名称+报表名（如"XX公司资产负债表"）
            m = re.search(r'(.+?)(?:资产负债表|利润表|现金流量表)', text)
            if m and len(m.group(1).strip()) > 4:
                wb.close()
                return m.group(1).strip()
            # 模式3：整行就是公司名称（不含"表"字）
            if not re.search(r'[表报]', text) and re.search(r'(公司|厂|集团|有限|合伙)', text):
                wb.close()
                return text
    wb.close()

    # 优先级2：科目余额表表头
    if kmye_path:
        wb = openpyxl.load_workbook(kmye_path, data_only=True)
        ws = wb.active
        for r in range(1, 6):
            cell_val = ws.cell(row=r, column=1).value
            if cell_val:
                text = str(cell_val).strip()
                if re.search(r'(公司|厂|集团|有限|合伙)', text):
                    wb.close()
                    return text
        wb.close()

    # 优先级3和4：由调用方从项目文件夹名/用户提供中提取
    return None
```

#### 0.7.2 提取评估基准日

```python
def extract_base_date(bs_path, kmye_path=None):
    """从财务报表中提取评估基准日"""
    # 优先级1：BS副标题/表头中的日期
    wb = openpyxl.load_workbook(bs_path, data_only=True)
    ws = wb.active
    for r in range(1, 8):
        for c in range(1, ws.max_column + 1):
            cell_val = ws.cell(row=r, column=c).value
            if cell_val:
                text = str(cell_val).strip()
                # 典型格式："2025年10月31日" / "2025年10月31" / "2025-10-31"
                m = re.search(r'(\d{4})\s*年\s*(\d{1,2})\s*月\s*(\d{1,2})\s*日?', text)
                if m:
                    wb.close()
                    return f"{m.group(1)}-{m.group(2).zfill(2)}-{m.group(3).zfill(2)}"
                m = re.search(r'(\d{4})-(\d{1,2})-(\d{1,2})', text)
                if m:
                    wb.close()
                    return f"{m.group(1)}-{m.group(2).zfill(2)}-{m.group(3).zfill(2)}"
    wb.close()

    # 优先级2：科目余额表表头中的截止日期
    if kmye_path:
        wb = openpyxl.load_workbook(kmye_path, data_only=True)
        ws = wb.active
        for r in range(1, 6):
            for c in range(1, ws.max_column + 1):
                cell_val = ws.cell(row=r, column=c).value
                if cell_val:
                    text = str(cell_val).strip()
                    m = re.search(r'(\d{4})\s*年\s*(\d{1,2})\s*月\s*(\d{1,2})\s*日?', text)
                    if m:
                        wb.close()
                        return f"{m.group(1)}-{m.group(2).zfill(2)}-{m.group(3).zfill(2)}"
                    m = re.search(r'(\d{4})-(\d{1,2})-(\d{1,2})', text)
                    if m:
                        wb.close()
                        return f"{m.group(1)}-{m.group(2).zfill(2)}-{m.group(3).zfill(2)}"
        wb.close()

    # 优先级3和4：由调用方从项目文件名/用户提供中提取
    return None
```

```python
import openpyxl

def fill_settings_sheet(detail_path, company_name, base_date):
    """在评估明细表的设定信息sheet中填写被评估单位和评估基准日"""
    wb = openpyxl.load_workbook(detail_path)
    ws = wb['设定信息']

    # B6 = 被评估单位全称
    if company_name:
        ws['B6'] = company_name

    # B7 = 评估基准日（YYYY-MM-DD格式）
    if base_date:
        from datetime import datetime
        try:
            # 尝试解析为datetime对象，确保格式正确
            dt = datetime.strptime(base_date, '%Y-%m-%d')
            ws['B7'] = dt
            ws['B7'].number_format = 'yyyy"年"m"月"d"日"'
        except ValueError:
            # 解析失败则直接写字符串
            ws['B7'] = base_date

    wb.save(detail_path)
    wb.close()

    # 验证填写结果
    wb2 = openpyxl.load_workbook(detail_path, data_only=True)
    ws2 = wb2['设定信息']
    b6_val = ws2['B6'].value
    b7_val = ws2['B7'].value
    wb2.close()

    return {
        'company_name': b6_val,
        'base_date': b7_val,
        'b6_filled': b6_val is not None and str(b6_val).strip() != '',
        'b7_filled': b7_val is not None and str(b7_val).strip() != ''
    }
```
