## 一、备抵科目体系

### 1.1 资产备抵科目（减值/折旧/摊销）

| 主科目编码 | 主科目名称 | 备抵科目编码 | 备抵科目名称 | 关系 | 评估明细表列示 |
|-----------|-----------|-------------|-------------|------|--------------|
| 1601 | 固定资产 | 1602 | 累计折旧 | 原值-折旧=净值 | 设备类Sheet双行表头 |
| 1601 | 固定资产 | 1603 | 固定资产减值准备 | 原值-折旧-减值=净值 | 减值行 |
| 1701 | 无形资产 | 1702 | 累计摊销 | 原值-摊销=净值 | 无形资产Sheet双行表头 |
| 1701 | 无形资产 | 1703 | 无形资产减值准备 | 原值-摊销-减值=净值 | 减值行 |
| 1122 | 应收账款 | 1231 | 坏账准备 | 余额-坏账=净额 | 坏账准备行(DT-151) |
| 1221 | 其他应收款 | 1231 | 坏账准备 | 余额-坏账=净额 | 坏账准备行(DT-151) |
| 1403/1405 | 存货 | 1471 | 存货跌价准备 | 余额-跌价=净额 | 跌价准备行 |

### 1.2 核心计算公式

```
固定资产：
  账面原值 = 1601借方余额
  累计折旧 = 1602贷方余额（取绝对值）
  账面净值 = 账面原值 - 累计折旧
  评估价值 = 账面净值（默认）

无形资产：
  账面原值 = 1701借方余额
  累计摊销 = 1702贷方余额（取绝对值）
  账面净值 = 账面原值 - 累计摊销
  评估价值 = 账面净值（默认）

应收款项：
  账面余额 = 1122/1221借方余额
  坏账准备 = 1231贷方余额（取绝对值）
  账面净额 = 账面余额 - 坏账准备
  明细表中：坏账准备行 I列=+|坏账|, J列=0 (DT-151)
```

### 1.3 科目余额表中的识别规则

- 1602(累计折旧)永远是**贷方余额**——如果出现借方余额，说明有冲回
- 1702(累计摊销)永远是**贷方余额**
- 1231(坏账准备)永远是**贷方余额**
- 备抵科目的balance为正数时，代表贷方余额（累计金额）

## 二、评估价值基本规则

### 2.1 默认规则

**评估价值 = 账面价值**（同符号同金额）

这是资产评估的默认假设：在无特殊评估方法（市场法、收益法等）适用时，评估价值等于账面价值。

### 2.2 例外规则

| 场景 | 账面价值 | 评估价值 | 规则来源 |
|------|---------|---------|---------|
| 坏账准备行 | +\|坏账金额\| | 0 | DT-151 |
| 预计风险行 | 0 | +\|减值评估值\| | DT-151 |
| 存货跌价准备行 | +\|跌价金额\| | 0 | 类比DT-151 |
| 固定资产(成本法) | 账面净值 | 重置全价×成新率 | 成本法评估 |
| 无形资产(成本法) | 账面净值 | 重置全价×成新率 | 成本法评估 |

### 2.3 严禁规则

- 🚨 **禁止** `评估价值 = abs(账面价值)` — 会导致符号不一致
- 🚨 **禁止** 负债类科目评估价值取正数 — 评估值与账面值必须同符号
- 🚨 **禁止** 凭空调整评估值 — 必须有评估方法支撑

## 三、科目方向与符号

### 3.1 基本规则

| 科目类别 | 正常方向 | 正数含义 | 负数含义 |
|---------|---------|---------|---------|
| 资产类(1xxx) | 借方 | 资产增加 | 资产减少/备抵 |
| 负债类(2xxx) | 贷方 | 负债增加 | 负债减少/借方余额 |
| 权益类(4xxx) | 贷方 | 权益增加 | 权益减少 |
| 成本类(5xxx) | 借方 | 成本增加 | 成本减少 |
| 损益类(6xxx) | 双向 | 视具体科目 | 视具体科目 |

### 3.2 评估明细表中的符号处理

- 所有金额**保留科目自然方向**（不做abs处理）
- 负债借方余额在明细表中显示为负数（正常现象）
- 资产贷方余额在明细表中显示为负数（如坏账准备）

### 3.3 应交税费特殊处理

应交税费(2221)的借方余额=增值税留抵税额：
- 在科目余额表中：direction='借'，balance=正数
- 在评估明细表中：账面价值=负数（负债方借方余额）
- 评估价值=账面价值（同负数）
- **不需要重分类**（前提：BS已对平）

## 四、重分类决策树

```
Step 1: 检查BS与科目余额表是否对平
  ├── 已对平 → 不做重分类，按科目余额表原始口径填写
  └── 未对平 → 进入Step 2

Step 2: 查找差异原因
  ├── BS有重分类调整 → 查BS附注确认重分类项
  ├── BS口径不同 → 按BS口径拆分
  └── 原因不明 → 标注差异，不自动重分类

Step 3: 执行重分类
  ├── 应交税费借方余额 → 3-13其他流动资产（仅BS有重分类时）
  ├── 应收/预收对冲 → 按BS口径
  └── 其他 → 按BS附注
```

## 五、去重规则

### 5.1 核心原则

- **仅按科目编码(code)去重**
- **禁止按名称(name)去重**
- 不同编码的同名资产是不同资产

### 5.2 典型案例

- 180101(办公室装修费) vs 18010802(办公室装修费)：编码不同=不同资产，两条都保留
- 同一供应商在不同地区有多个往来：编码不同=不同记录

### 5.3 dedup_key配置

- 往来科目：dedup_key=["name", "balance"] — 同名同金额才合并
- 长期待摊等：dedup_key=["code"] — 同编码才合并
- 永远不做纯name去重
