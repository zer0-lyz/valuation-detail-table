# 审核类Skill共享工作流框架

> 本文件汇总了所有审核类skill共用的Phase工作流框架。各skill在SKILL.md中引用本框架，仅保留领域特定的Phase步骤。

## 通用Phase框架

### Phase 0: 项目建档与工作底稿提取
1. 文件规模探测（Sheet数、文件大小）
2. 隐藏状态检测（T9强制，必须在数据提取之前）
3. Sheet结构概览提取
4. 基础信息提取（封面/设定信息）
5. 创建问题追踪文件

### Phase 1: 工作底稿结构梳理
1. 识别评估方法（成本法/收益法/市场法）
2. 梳理Sheet逻辑关系与数据流向
3. 识别关键参数单元格位置
4. 输出结构梳理报告

### Phase 2: 公式链接与数据一致性检查
1. 公式文本提取（data_only=False）
2. 计算值提取（data_only=True）
3. 公式与计算值交叉比对
4. 跨Sheet引用链路追踪
5. 异常值标记（#REF!、#VALUE!、#DIV/0!等）

### Phase 3: 验算与交叉核对
1. 关键参数独立验算
2. 中间环节独立验证（T5落地）
3. 跨Sheet数据交叉核对
4. 评估方法逻辑验证

### Phase 4: 评估参数合理性分析
1. 关键参数合理性判断
2. 可比数据/市场数据对比
3. 敏感性分析（如适用）
4. 参数选取依据充分性审核

### Phase 5: 特别事项与产权检查
1. 产权瑕疵识别
2. 特别事项说明完整性
3. 假设条件合理性
4. 限制条件合规性

### Phase 6: 汇总审核意见
1. 问题分级（🔴严重/🟡需核实/🟢建议）
2. 问题分类汇总
3. 章节完整性确认（T8）

### Phase 6.5: 复盘与经验转化（禁止跳过，T17）
1. 问题审视与漏检分类
2. 经验转化（T13落地）
3. 脚本归库（T18落地）
4. 脱敏合规扫描
5. 汇报复盘
6. 收敛

### Phase 7: 生成审核报告
1. 报告结构与章节完整性检查
2. 审核执行透明度附注
3. 交付（Phase 6.5自检清单全部"已执行"后方可）

## 通用质量保障机制

- 每个Phase结束时，自我验证该Phase所有检查项是否逐项执行完毕
- 每个Phase实际工作完成后，立即调用TaskUpdate标记completed
- 每个Phase的发现立即写入问题追踪文件
- 会话中断恢复：重启后先执行TaskList确认各Phase状态
- 审核人沟通中确认的改进信息，立即记录到审核追踪文件临时备注区

## 通用输出格式原则

- 交付时机：Phase 7 完成后
- 交付格式：Markdown（用户偏好），如有要求再转Word
- 禁止Phase 7完成前调用deliver_attachments
