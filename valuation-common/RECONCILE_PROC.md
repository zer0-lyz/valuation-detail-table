# RECONCILE_PROC.md — 跨skill校对共享过程

> 版本：v1.0 | 创建：2026-05-19
> 用途：定义如何从评估明细表/成本法底稿中读取指定科目的校对值，供各skill统一调用

---

## 1. 目标

在文件识别/整理/底稿编制过程中，将文件中提取的关键数值与评估明细表或成本法底稿中的对应值进行校对，及时发现并报告不一致。

---

## 2. 输入

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `subject_code` | ✅ | 底稿科目代码 | `C3-1-2` |
| `reconcile_items` | ✅ | 校对项列表 | `["bank_name", "account_no", "ending_balance"]` |
| `period` | ✅ | 校对期间 | `202512`（YYYYMM格式） |
| `detail_table_path` | ⚠️ | 评估明细表路径（与workpaper_path至少填一个） | — |
| `workpaper_path` | ⚠️ | 成本法底稿路径 | — |

---

## 3. 数据源优先级

```
成本法底稿过程表 > 评估明细表
```

- 成本法底稿过程表包含最细粒度的科目数据，优先读取
- 评估明细表为汇总级数据，作为备选/补充
- 若两者均可读取且值不一致，以成本法底稿为准，同时标注差异

---

## 4. 校对项定义（当前支持）

### 4.1 银行存款科目（C3-1-2）

| 校对项ID | 校对项名称 | 文件侧提取方式 | 明细表/底稿侧定位 |
|----------|-----------|---------------|-------------------|
| `bank_name` | 开户行名称 | PDF文本提取"开户行"字段 | 过程表对应银行行的"开户行"列 |
| `account_name` | 账户名称 | PDF文本提取"账户名称/户名"字段 | 过程表对应银行行的"账户名称"列 |
| `account_no` | 账号 | PDF文本提取"账号/活期账号"字段 | 过程表对应银行行的"账号"列 |
| `ending_balance` | 期末余额 | PDF文本提取"期末账户余额/本对账期末余额"字段 | 过程表对应银行行的"账面价值"列或明细表对应行 |

## 5. 读取规则

### 5.1 从成本法底稿读取

```python
# 定位逻辑（以银行存款为例）
# 1. 打开成本法底稿Excel
# 2. 定位sheet: 按 subject_code 匹配过程表名称
#    - 规则：过程表名通常包含科目名称关键字
#    - 示例：C3-1-2 → 定位"银行存款"过程表
# 3. 定位行: 按 reconcile_item 中的关键字匹配
#    - 示例：bank_name → 在"开户行"列查找
#    - 示例：account_no → 在"账号"列查找
# 4. 读取值: 返回对应单元格的值
```

### 5.2 从评估明细表读取

```python
# 定位逻辑
# 1. 打开评估明细表Excel
# 2. 定位sheet: "评估明细表"
# 3. 定位行: 按 subject_code 在科目代码列查找
# 4. 定位列: 按 reconcile_item 映射到列名
#    - ending_balance → "账面价值"列
```

### 5.3 银行存款特定逻辑

银行存款科目在成本法底稿中通常按银行分行列示（一行一个银行账户），需先按银行名称/账号匹配到具体行：

```
匹配优先级：账号 > 开户行名称 > 账户名称
```

## 6. 输出

```json
{
  "subject_code": "C3-1-2",
  "period": "202512",
  "items": [
    {
      "reconcile_item": "bank_name",
      "file_value": "中国银行杭州城西科创支行",
      "ref_value": "中国银行杭州城西科创支行",
      "ref_source": "成本法底稿-银行存款过程表-行3",
      "result": "✅一致"
    },
    {
      "reconcile_item": "ending_balance",
      "file_value": "36,307,755.38",
      "ref_value": "35,646,166.10",
      "ref_source": "评估明细表-银行存款行",
      "result": "❌不一致"
    }
  ],
  "overall_result": "❌存在不一致"
}
```

---

## 7. 不一致处理

| 情况 | 处理 |
|------|------|
| 一致 | 记录reconcile_result=✅，继续 |
| 不一致 | ⛔暂停，提请人工判断（MR-9异常即停），禁止AI自行推断原因或修改任何值 |
| 无对应科目 | 记录reconcile_result=⏭️跳过，注明原因 |
| 两个数据源均可读且值不一致 | 以成本法底稿为ref_value1、评估明细表为ref_value2，两者都报告 |

