## 共享规则总表

| # | 执行纪律 | 适用范围 | 违反后果 |
|---|---------|---------|---------|
| **T0** | **零幻觉原则**：每一条审核意见必须有底稿中的具体数据（单元格位置、数值、公式）作为支撑，禁止基于推测、假设、"应该是"做出判断 | Phase 0-6 | 任何基于幻觉的审核意见 = 审核质量事故 |
| **T1** | **每个Phase结束时必须输出自检清单**，逐项填写"已执行/不适用/跳过（原因）"，不得口头确认"已检查" | Phase 1-6 | 无自检清单输出 = 该Phase视为未完成 |
| **T2** | **标记"已执行"的检查项必须有Python脚本执行输出作为支撑** | Phase 2-5 | 无脚本输出却标记"已执行" = 审核造假 |
| **T3** | **所有数据提取必须同时使用`data_only=False`和`data_only=True`两种模式** | Phase 0, 2, 3 | 仅用data_only=True = 公式级错误不可见 |
| **T4** | **检查覆盖面必须100%**：不得以"代表性抽检""关键项检查"为由跳过任何Sheet/行/列 | Phase 1-5 | 覆盖面不足 = 漏检风险极高 |
| **T5** | **末端验证通过 ≠ 上游正确**：每个中间环节都必须独立验证，不得假设"下游结果正确则上游必然正确" | Phase 2, 3 | 仅做末端验证 = 中间过程完全不可控 |
| **T6** | **每个Phase的问题实时写入问题追踪文件**，不得等到Phase 6才汇总 | Phase 0-5 | 延迟记录 = 上下文丢失风险 |
| **T7** | **嵌套子流程必须有显式执行指令**，不得假设"写在文档里就会自动执行" | Phase 4, 6 | 嵌套子流程被跳过 = 关键结论缺失 |
| **T8** | **审核报告必须对照章节完整性清单逐项确认** | Phase 6 | 章节遗漏 = 报告质量不达标 |
| **T9** | **跳过隐藏内容**：隐藏的Sheet、行和列不参与任何审核环节。审核范围仅限于可见（未隐藏）的工作簿、工作表、行和列。**隐藏行列中的公式错误、#REF!等任何问题，均不纳入审核、不提出问题。** 判断方法：通过XML检查`hidden="1"`属性或`outlineLevel`属性确认隐藏状态，视觉展开大纲层级≠取消隐藏。**⚠️ 执行顺序强制约束：隐藏状态检测必须在数据读取之前执行。** 禁止先读取数据、发现异常后再回头检测隐藏状态。**⚠️ 隐藏状态必须实测确认，禁止假设：**含有#REF!的行不一定都是隐藏行，必须通过脚本逐一检测`row_dimensions[r].hidden`属性 | Phase 0-6 | 对隐藏内容提出审核意见 = 审核范围越界；先提意见再检测隐藏 = 执行顺序违规；假设含#REF!的行一定隐藏 = 审核范围误判 |
| **T10** | **大文件精准提取**：Sheet≥30或文件≥5MB时，必须使用read_only模式，WPS文件必须预设ZIP限制`zipfile.ZIP_MAX_COMMENT_LEN = 1000000` | Phase 0-5 | 全量加载超时 = 审核中断 |
| **T11** | **上下文容量管理**：脚本输出重定向到UTF-8文件，单次不超过500行 | Phase 0-6 | 控制台乱码+上下文溢出 = 会话断开 |
| **T12** | **脚本失败熔断**：1次失败立即换策略，禁止同一思路重试≥3次 | Phase 0-5 | 反复重试 = 上下文浪费 |
| **T13** | **漏检问题与审核人反馈强制转化为执行步骤**：复盘发现的漏检项和审核人指出的未发现问题，必须无条件转化为SKILL.md中的强制执行步骤（checkbox或自检行），零遗漏 | Phase 0-6.5 | 未转化 = 下次审核仍漏检 |
| **T15** | **保密脱敏强制纪律（全流程）**：skill目录下所有文件及子目录中，**严禁出现可识别项目/客户的信息**：①项目/客户/被评估单位真实名称→使用"项目A/B/C"编号；②底稿真实文件名→使用"某项目-底稿.xlsx"；③编制人姓名/工号→使用"某评估师"；④文件完整路径→使用"某路径/某文件.xlsx"；⑤脚本输出文件及目录名不得包含客户/项目名称；⑥可比公司真实名称在经验教训中使用"某可比公司"替代。**写入时即脱敏**，审核完成后必须删除scripts/下所有非.py运行产出文件，仅保留脚本本身 | 全Phase | 出现真实名称 = 保密违规 |
| **T16** | **任务状态同步**：每个Phase完成后立即调用TaskUpdate标记completed | 全Phase | 任务状态脱节 = 进度管理失控 |
